When we try to open a website in a browser,it does the work in some steps.First it converts the whole website in to a dom tree and style tree for css.Then it creats render tree.Then comes the js engine and this engine is named differently in defferent browsers.The javascript engine takes the human readable js code and converts it into machine readable code so that computer can understand our instuctions.And finally we can see the whole website in ui.

We know that,javascript is an interpreted ,high-level language.It runs the code line by line.But in the code ,sometimes it finds the code which is not necessary now.So it keeps those codes or function in a queue list.So when all the stacks are done,then it starts to run the code from the queue and creats an event loop.Setinterval and setTimeout are the best exaple of this thing.

Local storage:Local storage is available permanently.It doesn't remove automatically.We can remove this manually.If we close the browser or tab,and then come back still the local storage data will be stored there.
Session storage:Session storage is only available for the duration of browser session.When the browser or the tab is closed,session storage is gone automatically.
